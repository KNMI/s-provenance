  • CResource definition "Computational Resource where the workflow is submitted and executed" 
  • Component definition "The agent responsible to perform a task in the workflow. It represent an abstract structural element of the workflow and delegates to its instances the actual computation on the incoming data stream. Its Instances may be executed in parallel. More \emph{Component} agents cooperate in the execution of a \emph{Workflow}" 
  • ComponentParameters definition "Each \emph{ComponentInstance} and therefore their \emph{Invocations} are parametrised. Parameters' vaues gets assigned at the time of the configuration of the workflow execution, but can change at runtime." 
  • Data definition "Any \emph{Collection} of data produced or used by an \emph{Invocation}. Each element gets time-stamped and if not volatile, it can refer to the \emph{url} of its location. It is similar to the Collection entity used by PROV and also by PROV-ONE, however it has been redefined to associate more specific attributes to describe general characteristics of the output data produced; eg. location, format, size. S-PROV assumes for practical reasons that outputs should be always considered like collections of one or more granules." 
  • DataGranule definition "A \emph{DataGranule} is part of a \emph{Data} collection and contains contextual metadata related to the specific research topic and domain. Metadata can be associated with data formats and community schemas, as well as user-defined. The latter fosters the adoption of provenance for the rapid cataloguing and exploration of experimental properties that are extracted from intermediate results." 
  • Implementation definition "A Plan representing an atomic operation constituting a workflow. It is described by its source code or by a link or to an external versioning system. If applicable, it also indicates information about its type in the context of the native programming language. This is a subclass of \emph{provone:Process} therefore it has properties indicating input and outport ports and the traceability of its evolution." 
  • Invocation definition "The \emph{Activity} performed by a \emph{ComponentInstance} over incoming data. Each \emph{Invocation} can produce more \emph{Data}. It uses a collection of \emph{ComponentParameters}." 
  • StateCollection definition "The internal state of a \emph{ComponentInstance}. Contains references to \emph{Data} that are preserved by \emph{Invocations} during stateful operations." 
  • SystemProcess definition "Represents the Location where a ComponentInstance is allocated in a runtime-environment. It describes information about the computational worker node of a cluster and the Operative System process. Multiple ComponentInstances can be allocated to the same worker and also to the same process." 
  • WFExecution definition "The Activity representing the execution of a workflow run. From this activity qualified associations to agents such as users and software components are established. The class present also details about the execution environment." 
  • WFExecutionBundle definition "The Bundel representing the workflow execution setup. The related Entity presents attribution information and details of the execution environment." 
  • WFExecutionInput definition "The Collection of entities used for the general parametrisation of a WFExecution. The Collection can contain or refer to entities of different kinds; eg. data and configuration files, archive files containing the workflow implementation scripts, references to other workflow executions etc. The latter can indicate the exploitation of information produced by other workflows' runs, which indicates inter-workflow exchange."
